| **Total audited**                                              | 92 | 100% |

**What drove the single-pattern result:** every MINOR POLISH hit is the same anti-pattern — 
explicit chain-of-thought scaffolding ("think through X step by step before ...") that Opus 4.7 
performs automatically via adaptive thinking. Per the 4.7 release docs, explicit CoT prompts are 
redundant on 4.7 and can degrade performance by forcing reasoning even when the task doesn't need it.

**What was checked but not found:**

- Self-verification scaffolding (*"double-check your answer"*) — 11 weak matches, all false positives: they were content specs asking Claude to *produce* verification steps for the user's output, not scaffolding directed at Claude's own reasoning. None warranted an edit.
- Progress-update scaffolding (*"give me updates as you work"*) — 0 hits.
- Aggressive all-caps scaffolding (*CRITICAL:*, *MANDATORY:*, triple-caps shouting) — 0 hits after filtering bracketed template placeholders and inline negatives like *"Do NOT invent URLs"*.
- Emoji / warmth-tone requests that 4.7 would ignore — 0 hits.
- Generalization risk (*"and other relevant"*, *"etc."*) — present but never in isolation; all appeared alongside explicit enumeration, so 4.7 won't need to guess.

**Net effect:** the Prompt Hub library is in strong shape for Opus 4.7. 89% of variants ship as-is; the 10 drafts below each remove a single sentence of now-redundant reasoning scaffolding.

## Next steps for Craig

1. **Review the 10 drafts in Supabase.** Query:
   ```sql
   SELECT v.id, p.title, v.parent_variant_id, v.status, v.audit_source,
          length(v.prompt_text) AS new_len
   FROM prompt_hub_variants v
   JOIN prompt_hub_prompts p ON p.id = v.prompt_id
   WHERE v.audit_source = 'opus-4-7-audit-2026-04-18'
   ORDER BY v.id;
   ```
   Spot-check the `prompt_text` on each. Every draft is exactly one sentence shorter than its parent (or for id=105/321/333 has the scaffold sentence replaced with a direct version of the same guidance).

2. **Promote the drafts when you're satisfied.** Single-transaction flip (per parent→draft pair):
   ```sql
   BEGIN;
   UPDATE prompt_hub_variants SET status='superseded', superseded_by=:new_id WHERE id=:parent_id;
   UPDATE prompt_hub_variants SET status='active', last_verified_at=now() WHERE id=:new_id;
   COMMIT;
   ```
   The partial unique index `prompt_hub_variants_prompt_id_model_name_active_key` (added during this audit) enforces "one active variant per (prompt_id, model_name)" but only for `status='active'` rows, so draft and superseded rows coexist cleanly.

3. **Bulk-promote option.** If you want to flip all 10 at once:
   ```sql
   BEGIN;
   UPDATE prompt_hub_variants p
     SET status='superseded',
         superseded_by=d.id
     FROM prompt_hub_variants d
    WHERE d.audit_source='opus-4-7-audit-2026-04-18'
      AND d.status='draft'
      AND p.id = d.parent_variant_id;

   UPDATE prompt_hub_variants
      SET status='active', last_verified_at=now()
    WHERE audit_source='opus-4-7-audit-2026-04-18'
      AND status='draft';
   COMMIT;
   ```

4. **Roll back if needed.** To revert a single promotion:
   ```sql
   BEGIN;
   UPDATE prompt_hub_variants SET status='draft'       WHERE id=:new_id;
   UPDATE prompt_hub_variants SET status='active', superseded_by=NULL WHERE id=:parent_id;
   COMMIT;
   ```

5. **Rebuild `prompts.alignai.business` after promotion.** `build.py` reads `v_prompt_variants_full`, which filters to `pv.status = 'active'`. After you flip a draft to active, the next site build will pick it up automatically. Drafts are never exposed to the public view.

6. **Refresh `last_verified_at` and bump `model_version`.** Set `model_version='Claude 4.7'` on promotion so the Prompt Hub freshness badges reflect this audit:
   ```sql
   UPDATE prompt_hub_variants
      SET model_version='Claude 4.7'
    WHERE audit_source='opus-4-7-audit-2026-04-18' AND status='active';
   ```

7. **Applied schema changes (kept).**
   - Added columns to `prompt_hub_variants`: `status`, `parent_variant_id`, `audit_source`, `superseded_by` (+ `idx_variants_status`).
   - Updated view `v_prompt_variants_full` to filter `pv.status = 'active'`.
   - Replaced the global `UNIQUE(prompt_id, model_name)` constraint with the partial unique index `prompt_hub_variants_prompt_id_model_name_active_key … WHERE status='active'` so draft/superseded rows can coexist.
   All three changes are permanent and safe to leave in place for future audits.
